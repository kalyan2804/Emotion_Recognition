Abstract

Sentiment analysis is a technique used to analyse different people's attitudes, feelings, and opinions towards anything, and it may be performed on tweets to analyse public opinion on news, legislation, social movements, and individuals. Twitter, as a microblogging site, is being used to collect opinions on products, trends, and politics. Opinion mining can be accomplished without manually reading tweets by using Machine Learning models. Their findings could help governments and corporations implement policies, products, and events. Machine Learning algorithms are used to recognise emotions in tweets by categorising them as happy or unhappy. An in-intensity comparative overall performance research found out that the proposed vote casting classifier (LR-SGD) with TF-IDF yields optimal results.

        A decision tree is a decision support tool that uses a tree-like model of decisions and their possible consequences, including chance event outcomes, resource costs, and utility. It is one way to display an algorithm that only contains conditional control statements. A decision tree is a flowchart-like structure in which each internal node represents a "test" on an attribute (e.g. whether a coin flip comes up heads or tails), each branch represents the outcome of the test, and each leaf node represents a class label (decision taken after computing all attributes). The paths from root to leaf represent classification rules. Decision Trees (DTs) are a non-parametric supervised learning method used for classification and regression. The goal is to create a model that predicts the value of a target variable by learning simple decision rules inferred from the data features. Decision tree provides more efficiency than the existing system.

3. Sourcecode:

#! /usr/bin/python2

import pandas as pd  #Used for data analysis

import numpy as np   #Used for performing numerical operations on Arrays

import pickle  

import sklearn.ensemble as sk

from sklearn import model_selection #Helps in splitting the data into Train and Test Dataset

from sklearn import tree  #Importing Tree module for accessing DecisionTree Classifier

from sklearn import linear_model

from sklearn.feature_selection import SelectFromModel #Helps in selecting features from the Dataset

import joblib

from sklearn.metrics import confusion_matrix #Confusion Matrix gives the Count of predicted and actual values

data_frame = pd.read_csv('data.csv', sep='|',encoding= 'latin1') #Loading the dataset

x = data_frame.drop(['Name', 'md5', 'legitimate'], axis=1).values #Selecting input fetures

y = data_frame['legitimate'].values #Output feature

print("Extracted important features from the total of %i Features\n"% x.shape[1])

feature_sel = sk.ExtraTreesClassifier().fit(x, y)  #Selecting important features based on

model = SelectFromModel(feature_sel, prefit=True)

x_new = model.transform(x)

new_features = x_new.shape[1]

#Splitting the dataset into train and test sets in the ratio 80:20

x_train, x_test, y_train, y_test = model_selection.train_test_split(x_new, y ,test_size=0.2)

features = []

print("Number of features identified as important : %i" % new_features)

indices = np.argsort(feature_sel.feature_importances_)[::-1][:new_features]

for f in range(new_features):

    print("%d. feature %s (%f)" % (f + 1, data_frame.columns[2+indices[f]], feature_sel.feature_importances_[indices[f]]))

for f in sorted(np.argsort(feature_sel.feature_importances_)[::-1][:new_features]):

    features.append(data_frame.columns[2+f])

#Training the model with  DecisionTreeClassifier

algorithms = {

        "DecisionTree": tree.DecisionTreeClassifier(max_depth=10),

    }

results = {}

print("\nNow Testing the Algorithm")

for alg in algorithms:

    clf = algorithms[alg]

    clf.fit(x_train, y_train)

    score = clf.score(x_test, y_test)

    print("%s : %f %%" % (alg, score*100))

    results[alg] = score

best = max(results, key=results.get)

print("\n%s with a %f %% success" % (best, results[best]*100))

print("Algorithm saved in Classifier Directory...")

joblib.dump(algorithms[best], 'classifier/classifier.pkl')

open('classifier/features.pkl', 'wb').write(pickle.dumps(features))

print("Saved...")

#Predicting outcomes for the test dataset

clf = algorithms[best]

res = clf.predict(x_test)

mtrx = confusion_matrix(y_test, res)

print("False Positive rate : %f %%" % ((mtrx[0][1] / float(sum(mtrx[0])))*100))

print('False Negative rate : %f %%' % ((mtrx[1][0] / float(sum(mtrx[1]))*100)))

5. Conclusion

•A novel combination of LR and SGD as a voting classifier for emotion recognition by classifying tweets as happy or unhappy is developed.

Emotion recognition by classifying the text data as happy or unhappy is developed by using the novel combination of LR & SGD as a voting classifier.

•Our investigations revealed that by swiftly recognising patterns and effectively averaging combinations of models, one can improve the performance of models.

Through our observations, we can conclude that one can improve the performance of models by recognising the hidden patterns and effective averaging of combinations of models.

•Experiments are conducted to test seven machine learning models that are; (1) SVM, (2) RF, (3) GBM, (4) LR, (5) NB and (6) VC(LR-SGD).

The observations include seven learning models, i.e., LR, GBM, NB, RF, SVM, SGD & VC LR-SGD.

•All models performed well on the twitter dataset, however our proposed voting classifier VC(LR-SGD) outperformed them all by using both TF and TF-IDF.

By using TF & TF-IDF, the proposed model gave optimal results than all the other models on the twitter dataset.

•Proposed VC model achieves the highest results using TF-IDF with 80.6% Accuracy, 84.9% Precision ,74.5% Recall and 79.4% F1-score.

With 80.6% Accuracy, 84.9% Precision, 74.5% Recall & 79.4% F1-Score, our proposed Voting Classifier (LR-SGD) obtains the highest results.
